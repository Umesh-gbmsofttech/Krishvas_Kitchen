## How The App Works

### 1. First Launch
- Splash screen opens.
- App checks if a newer app version is available on GitHub Releases.
- If update is available, user sees a popup with a download button.

### 2. User Flow
- User can browse menu items without login.
- Login/signup is required for:
- placing orders
- viewing order history
- delivery partner registration
- User opens daily menu, checks item details, adds items to cart, and completes checkout.
- User chooses payment mode (COD or UPI).
- Order ID is generated after order placement.
- User tracks order status and delivery movement in tracking screen.

### 3. Admin Flow
- Admin logs in and opens admin dashboard.
- Admin creates menus, edits menus, and schedules menus (up to 30 days).
- Admin reviews and manages orders.
- Admin reviews delivery partner requests and approves/rejects.
- Admin checks reports and user list.

### 4. Delivery Partner Flow
- Any user can apply from profile using "Become a Delivery Partner".
- Admin receives request and approves/rejects.
- After approval, delivery badge is shown in profile.
- Delivery partner gets dashboard with:
- today's deliveries
- last 7 days
- last 30 days
- distance covered
- Delivery partner updates live location from map view for active deliveries.

### 5. Notifications
- Users, admins, and delivery partners receive real-time updates:
- new order alerts
- order status changes
- delivery assignment
- new delivery partner request
